How Cryptographic Hashing Works

What is a Hash Function?

A cryptographic hash function takes an input (e.g., text, files, passwords) of arbitrary length and returns a fixed-length string of bytes. It is a one-way function: it is computationally impossible to reverse-engineer the original input from the output hash.

Common algorithms include:

  • MD5: 128-bit hash. Extremely fast but considered cryptographically broken due to collision vulnerabilities.
  • SHA-1: 160-bit hash. Also deprecated for critical systems, but still seen in legacy platforms.
  • SHA-256: 256-bit hash. Member of the SHA-2 family. Highly secure and widely used in modern authentication, SSL certificates, and Bitcoin.
Dictionary Attacks vs. Brute-Force Attacks

Because hashes are irreversible, attackers use guessing techniques:

  • Dictionary Attacks: The cracker hashes list items from pre-compiled collections of commonly used passwords (like RockYou.txt or 10k-most-common) and checks if the generated hash matches the target. Fast and highly successful because humans choose predictable passwords.
  • Brute-Force Attacks: The cracker systematically computes hashes for every possible character combination (e.g., aaaaa, aaaab, aaaac) until a match is found. Computationally expensive and impractical for long, complex passwords.
Salting: Defeating Pre-computed Rainbow Tables

If users share passwords, or use common ones, attackers can compute the hashes of millions of common passwords in advance. These databases of pre-computed hashes are called Rainbow Tables.

To defeat this, databases store passwords with a Salt. A salt is a random string added to the password before hashing:
hash = SHA-256(password + salt)

Even if two users share the same password "123456", they get unique salts, creating completely different hashes. Rainbow tables are rendered completely useless!
